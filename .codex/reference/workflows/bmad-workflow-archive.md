# BMAD 归档管理工作流

## 🎯 归档目标

为已完成的项目或任务创建完整的归档记录，确保项目成果得到妥善保存和管理。

## 📋 归档流程

### 第一阶段：项目状态确认
1. **检查项目完成状态**
   - 确认所有任务已完成
   - 验证代码质量
   - 检查文档完整性

2. **收集项目信息**
   - 项目名称和描述
   - 实施时间线
   - 技术栈信息
   - 成果统计

### 第二阶段：创建归档记录
1. **创建内存归档**
   - 使用 create_memory 工具
   - 记录完整项目信息
   - 包含文件架构和技术成就

2. **生成归档报告**
   - 项目总结报告
   - 技术成就清单
   - 使用指南和最佳实践

### 第三阶段：验证和确认
1. **验证归档完整性**
   - 检查归档信息完整性
   - 验证文件路径正确性
   - 确认技术细节准确

2. **确认归档状态**
   - 更新项目状态为"已归档"
   - 记录归档时间
   - 设置维护状态

## 🔧 归档工具

### 内存归档工具
```typescript
// 使用 create_memory 工具
createMemory({
  Action: 'create',
  Content: '完整的项目归档信息',
  CorpusNames: ['project-name'],
  Tags: ['archive', 'completed', 'project-name'],
  Title: '项目归档标题',
  UserTriggered: true
})
```

### 文件验证工具
- 检查关键文件存在性
- 验证文件内容完整性
- 确认文档结构正确

## 📊 归档内容标准

### 必需内容
- [x] 项目概览和目标
- [x] 实施成果和统计
- [x] 完整文件架构
- [x] 技术成就总结
- [x] 使用指南
- [x] 维护信息

### 可选内容
- [ ] 性能指标
- [ ] 用户反馈
- [ ] 改进建议
- [ ] 未来规划

## 🎯 归档完成标准

### 质量标准
- **完整性**: 所有必需内容都已包含
- **准确性**: 技术信息和统计数据准确
- **可读性**: 文档结构清晰，易于理解
- **可检索性**: 信息易于搜索和查找

### 功能标准
- **快速检索**: 能够快速找到项目信息
- **状态追踪**: 项目状态清晰记录
- **维护支持**: 提供维护和更新指导
- **参考价值**: 为未来项目提供参考

## 📞 归档后维护

### 定期检查
- 每季度检查归档信息准确性
- 更新技术栈版本信息
- 补充新的最佳实践

### 持续改进
- 收集用户反馈
- 优化归档流程
- 改进文档质量
