# `@sprint` Compatibility Guide

`@sprint` is a copied alias from older BMAD planning material.

In this repository, sprint-style planning is not a separate user-facing mode.

## Local Mapping

If you need milestone or sprint-style planning, use:

- `.codex/workflows/bmm.md`
- `.codex/core/work-breakdown.md`
- `.codex/core/milestone-design.md`

The copied upstream sprint-planning wrappers remain reference material only.
