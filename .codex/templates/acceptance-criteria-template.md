# Acceptance Criteria Template

Use this when a task or milestone needs explicit pass/fail conditions.

```md
## Acceptance Criteria

- [ ] behavior: <observable outcome>
- [ ] scope boundary: <what must be included>
- [ ] non-goal preserved: <what must not change>
- [ ] validation: <command, test, or manual proof>
- [ ] fallback: <safe stopping point if partial>
```
