
# ✅ On-Page SEO 发布前检查清单 (模块 B)

**目标:** 确保每一篇内容（博客、文档、落地页）在上线前都经过完美的页面优化。将此清单作为通用的质量门禁。

---

### **阶段一：核心元数据与身份 (清单 #21-27, 29-30)**

-   [ ] **URL 规范:** URL 是否简短、小写、使用连字符 (`-`)，并包含主关键词？ (例如: `/[category]/[primary-keyword]`)
-   [ ] **Title 标签:** `<title>` 长度是否在 50-60 字符之间，主关键词是否尽量靠前？
-   [ ] **Meta Description:** Meta 描述长度是否在 150-160 字符之间，是否包含主关键词和吸引点击的 CTA？
-   [ ] **H1 标签:** 页面上是否有且仅有一个 `<h1>` 标签，且包含主关键词？
-   [ ] **图片 Alt 文本:** 所有图片是否都有描述性的 `alt` 文本，并自然地融入了相关关键词？
-   [ ] **图片命名:** 所有图片文件名是否都具有描述性并使用连字符？ (例如: `[feature]-dashboard-screenshot.webp`)

---

### **阶段二：内容结构与链接 (清单 #28, 31-37)**

-   [ ] **标题层级:** `<h2>` 和 `<h3>` 标签是否逻辑清晰地构建了内容结构，并包含了长尾关键词？
-   [ ] **关键词布局:** 主关键词是否自然地出现在文章的前 100 个词内？
-   [ ] **内部链接:** 是否有 5-10 个相关的内部链接指向其他支柱页或集群文章？
-   [ ] **锚文本:** 内链的锚文本是否具有描述性（例如“了解我们的 [Topic] 策略”），而不是“点击这里”？
-   [ ] **外部链接:** 是否至少有一个链接指向高权威、非竞争的外部网站。

---

### **阶段三：E-E-A-T 与转化 (清单 #38-45)**

-   [ ] **E-E-A-T 证明:** 内容是否展示了真实的经验、专业性、权威性和可信度？（例如，通过原创数据、案例研究或个人经历）
-   [ ] **主 CTA:** 在页面“首屏”（无需滚动即可见）是否有一个清晰、高可见度的转化行动按钮 (Call-to-Action)？
-   [ ] **FAQ 区块:** 如果适用，页面是否包含一个 FAQ 部分（并使用 `FAQPage` schema）来回答相关的长尾问题？
-   [ ] **发布/更新日期:** 是否清晰地显示了“最后更新时间”以向用户和搜索引擎表明内容的新鲜度？
