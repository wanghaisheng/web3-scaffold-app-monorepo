
# ✍️ Growth Spec: Copywriting Optimization Strategy

**Core Doctrine:** "Visuals by Drawing, Logic by Speaking, Launch Globally."

## 1. The Strategy: From "Command" to "Ship"
We align all copy to a concrete, actionable workflow. The user is not buying "management"; they are buying "results" (a shipped product).

### Key Pivot
*   **Old Hook:** "Stop Coding. Start Commanding." (Abstract)
*   **New Hook:** "Turn Your AI Sketch into Global SaaS." (Concrete)
*   **Reason:** Users want to know *what* they get. A "Global SaaS" is a valuable asset. "Commanding" is just work.

## 2. The G+L+C Narrative Structure
Every piece of marketing copy must reinforce the three-step ritual.

### G: Google AI Studio (Visuals)
*   **Keywords:** "Vision", "Sketch", "Free Brain", "Pixel Perfect".
*   **Pain Point:** "I can imagine it, but I can't code it."
*   **Resolution:** Upload a screenshot -> Get Code.

### L: Lovable (Logic)
*   **Keywords:** "Spine", "Backend", "Database", "Payments", "Natural Language".
*   **Pain Point:** "I have a pretty frontend, but it does nothing."
*   **Resolution:** "Connect this form to Supabase" -> Done.

### C: Cloudflare (Launch)
*   **Keywords:** "Body", "Global Edge", "Zero Config", "Instant".
*   **Pain Point:** "Deploying is hard. Servers are expensive."
*   **Resolution:** Git Push -> Live Worldwide.

## 3. Tone of Voice Rules
1.  **Be the "Old Dog"**: Use the persona of an experienced veteran who found a shortcut. It builds trust.
2.  **Attack the "Maintenance Trap"**: Constantly remind users of the pain of local environments (npm install, version conflicts).
3.  **Celebrate "Cheating"**: Frame efficiency as an unfair advantage or arbitrage. "Why code when you can speak?"

## 4. CTA Optimization
*   **Top Funnel:** "Get The Protocol" (Curiosity)
*   **Bottom Funnel:** "Start Shipping" (Action)
*   **Avoid:** "Learn More" (Passive)

## 5. The "Customer Language" Protocol (Feature Translation)
**Rule:** Communicate in "Customer Language", NOT "Feature Lists".
**Objective:** Content must be organized around "What problem does this solve?" or "Customer Success Stories", rather than listing technical specifications.

| ❌ Technical Feature (Internal) | ✅ Customer Language (External) |
| :--- | :--- |
| "React 19 + TanStack Router + Vite" | "Ship a high-performance app without configuring a server." |
| "Automated JSON-LD Schema Generation" | "Rank on Google automatically without hiring an SEO agency." |
| "Supabase Row Level Security" | "Your user data is safe and enterprise-ready." |

*   **Social Proof Priority:** Showing recommendations from other builders (e.g., "How Coach X saved 20 hours") is more effective than listing 50 features.
*   **Structure:** Every feature block must be paired with a specific **"Pain Relieved"** or **"Result Achieved"**.
