
# 📈 关键词研究协议与内容简报模板 (模块 A)

**目标:** 系统性地识别高意图关键词，为 Topic Cluster 内容策略奠定数据基础，严格遵循模块 A 的20条清单要求。

---

## 阶段一：种子关键词识别 (清单 #1-5)

这是所有研究的起点。识别与你的核心业务/产品最直接相关的关键词。

1.  **明确核心业务线**: 列出你的主产品、核心功能和服务。
2.  **定义 Money Keywords**: 为每个能直接带来收入的功能定义高商业意图的关键词（例如“react deploy tool”）。
3.  **按意图分类**:
    *   **学习型 (Informational):** “what is generative engine optimization”
    *   **比较型 (Commercial):** “vercel vs cloudflare”
    *   **购买型 (Transactional):** “instant ship pricing”
4.  **建立关键词分组**: 围绕每个产品创建“对比(vs)”、“替代(alternative)”、“用例(use case)”、“定价(pricing)”等关键词矩阵。

**交付物:** 一个包含 5-10 个核心种子关键词的清单。

---

## 阶段二：关键词扩展 (清单 #6-10)

使用专业工具和社区洞察，将种子关键词扩展为一个庞大的关键词池。

1.  **工具挖掘**: 使用 Semrush/Ahrefs/GSC 导出与种子关键词相关的所有长尾词和问题词。
2.  **数据标注**: 为每个关键词标注**搜索量 (Volume)**、**难度 (KD)**、**CPC** 和**主要国家**。
3.  **竞品分析**: 导出 10-20 个主要竞争对手网站的核心排名关键词，寻找他们的流量来源。
4.  **社区洞察 (Reddit/论坛)**: 搜索 "how to...", "best alternative for..." 等句式，挖掘用户真实、自然的提问方式。

**交付物:** 一个包含数百个关键词的 master spreadsheet (Excel/Google Sheets)。

---

## 阶段三：Topic Cluster 规划 (清单 #11-19)

将关键词组织成“支柱-集群”模型，构建网站的内容护城河。

1.  **定义支柱主题**: 围绕产品核心能力，定义 5-10 个核心的 Topic Cluster (例如“AI-Native SEO”, “React Performance”)。
2.  **映射集群关键词**: 将关键词池中的长尾词和问题词，分别归类到每个 Topic Cluster 下。
3.  **规划页面类型**: 明确每个关键词需要哪种页面来承载：
    *   **支柱页 (Pillar Page):** 对应核心主题，做深度综述。
    *   **集群页 (Cluster Page):** 对应长尾词，做具体问题的解答。
    *   **金钱页 (Money Page):** 对应高商业意图词，如对比页、替代页。
4.  **建立内链规划**: 在集群页的规划中，预先定义好指向支柱页的内链锚文本。

**交付物:** 一份 Topic Cluster 规划图或思维导图，明确了支柱页与集群页的层级关系和内链结构。

---

## 阶段四：内容简报生成 (清单 #20)

为每一个待创建的内容页面，生成一份标准化的内容简报 (Content Brief)，指导作者或 AI 进行创作。

**交付物模板 (Content Brief Template):**

```markdown

- **👤 目标受众:** [例如：寻求 Vercel 替代方案的独立开发者]
- **🔑 主要关键词:** [一个]
- **🗝️ 次要/长尾关键词:** [3-5个]
- **🕵️ 搜索意图 (Intent):** [例如：用户希望看到详细的功能对比、价格差异和性能基准测试]
```
