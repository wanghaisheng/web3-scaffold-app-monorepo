
# 模板: 高意图“金钱页面”模板 (Money Page Templates)

使用这些模板来创建针对漏斗底部的用户、具有高转化率的页面。

---

## 1. 竞品对比页 (X vs Y)

**URL:** `/compare/[competitor-slug]`
**主要关键词:** `[Your Product] vs [Competitor]`

```markdown
# [Your Product] vs [Competitor]: 哪个更适合 [特定用例]?

**[摘要段落]:** 一个快速、顶层的总结，为目标受众宣布胜利者。例如：“虽然 [Competitor] 对企业团队来说是个不错的选择，但对于优先考虑速度和成本效益的独立开发者和初创公司来说，**[Your Product] 无疑是赢家**。”

## 快速概览：功能对比

| 功能 | [Your Product] | [Competitor] |
| :--- | :--- | :--- |
| **目标用户** | ✅ 独立开发者, 初创公司 | ❌ 企业团队 |
| **定价模型** | 一次性/按量付费 | 高昂的月度订阅 |
| **核心优势** | 速度 / 零配置 | 遗留功能 |

## 深度解析：为什么 [Your Product] 胜出

### 1. [你最强的特性]
(详细解释为什么你的特性更优越，并附上截图。)

### 2. [对手最大的弱点]
(解释你的产品如何解决了对手造成的痛点。)

## 何时选择 [Competitor]?
(诚实一点，这能建立信任。例如：“如果你需要 [某个特定的遗留功能] 并且预算充足，[Competitor] 可能是一个合适的选择。”)

## 最终结论
(重申为什么你的产品是目标读者的更好选择。)

**[行动号召按钮: "免费开始构建"]**
```

## 2. 替代品页 (Alternative to X)

**URL:** `/[competitor]-alternative`
**主要关键词:** `Best [Competitor] Alternative`

```markdown
# [年份] 年最适合 [目标用户] 的 [Competitor] 替代品

**[介绍段落]:** 承认用户的痛点。例如：“您是否因为 [Competitor] 的高昂价格和复杂设置而寻找替代方案？您来对地方了。”

## 为什么用户会寻找 [Competitor] 的替代品

-   **痛点 1:** 成本高昂
-   **痛点 2:** 学习曲线陡峭
-   **痛点 3:** 不适合 [特定用例]

## 介绍 [Your Product]: 更优越的替代方案

以下是 **[Your Product]** 如何直接解决这些问题：

-   **解决痛点1:** 我们的定价是...
-   **解决痛点2:** 我们的设置仅需2分钟...

**[行动号召: "观看在线演示"]**
```

## 3. 用例页 (Use Case Page)

**URL:** `/for/[target-audience]`
**主要关键词:** `[Your Product] for [Target Audience]` (例如, "React Starter for Designers")

```markdown
# [Your Product]: 为 [目标用户] 打造的终极 [解决方案]

**[首屏英雄区]:** 一个直接与受众身份和主要目标对话的标题。例如：“从 Figma 到实时原型，只需几小时，而非几周。”

## 挑战: [受众] 最大的瓶颈
(描述这个受众面临的具体问题。例如：“作为一名设计师，你创造了精美的模型，但将其交给工程部门总是导致无休止的来回沟通和‘信息丢失’。”)

## [Your Product] 如何解决这个问题

**步骤 1: [特性 A]** - 解决问题的一部分。
**步骤 2: [特性 B]** - 解决另一部分。

**[来自该受众的客户评价]**
> "[成功使用你产品的设计师的引言]。"
> – [姓名], [职位] at [公司]

**[最终行动号召: "立即构建你的第一个原型"]**
```
