# 🧠 Product Spec: Requirement Analysis Protocol

**Context:** transforming a user request or PRD into actionable code tasks.

## Phase 1: The Backbone (Narrative)
1.  **Identify Actors**: Who is doing this? (Admin, User, System).
2.  **Map Journey**: Chronological flow (Sign Up -> Create -> Publish).

## Phase 2: The Slices (Release Strategy)
1.  **MVP Slice**: What is the absolute minimum to validate the value prop?
2.  **V2 Slice**: Nice-to-haves and optimizations.

## Phase 3: User Story Definition
Format: "As a [Actor], I want [Action], so that [Value]."
*   **Constraint**: Must map to a specific Route or Component in the scaffold.
*   **Data Requirement**: Define what CRUD operations are needed.
