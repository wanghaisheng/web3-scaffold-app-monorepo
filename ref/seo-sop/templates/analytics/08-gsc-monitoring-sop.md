> 每周监控 GSC，及时发现索引问题、抓取错误和排名变化。

## 📊 每周检查清单

### 1. 性能概览（5分钟）

#### 进入路径
GSC → 效果 → 搜索结果

#### 检查指标
| 指标 | 关注点 | 异常阈值 |
|------|--------|----------|
| 总点击数 | 周环比变化 | 下降 > 20% |
| 总展示次数 | 周环比变化 | 下降 > 20% |
| 平均 CTR | 保持稳定 | 下降 > 1% |
| 平均排名 | 保持或上升 | 上升 > 3位 |

#### 操作流程
```markdown

| 周 | 点击数 | 展示数 | CTR | 平均排名 | 周环比 | 备注 |
|----|--------|--------|-----|----------|--------|------|
| W01 | 5,000 | 100,000 | 5.0% | 12.5 | - | 基准 |
| W02 | 5,200 | 105,000 | 4.9% | 12.3 | +4% | 正常 |
```

### 2. 索引状态（5分钟）

#### 进入路径
GSC → 索引编制 → 页面

#### 检查项目
- [ ] 已编入索引的页面数量
- [ ] 未编入索引的页面数量
- [ ] 索引状态变化趋势

#### 未编入索引原因分析
| 原因 | 严重程度 | 处理方式 |
|------|----------|----------|
| 已抓取 - 目前尚未编入索引 | 🟡 中 | 等待或提交检查 |
| 已发现 - 目前尚未编入索引 | 🟡 中 | 提交索引请求 |
| 被 robots.txt 屏蔽 | 🔴 高 | 检查 robots.txt |
| 标记为 noindex | 🔴 高 | 检查 meta 标签 |
| 重复内容 | 🟡 中 | 设置 canonical |
| 404 | 🔴 高 | 修复或 301 重定向 |

### 3. 核心网页指标（3分钟）

#### 进入路径
GSC → 体验 → 核心网页指标

|------|------|------------|------------|
| LCP | ≤ 2.5s | ✅/❌ | ✅/❌ |
| INP | ≤ 200ms | ✅/❌ | ✅/❌ |
| CLS | ≤ 0.1 | ✅/❌ | ✅/❌ |

### 4. 安全与手动操作（2分钟）

#### 进入路径

#### 检查项目
- [ ] 无手动处置措施
- [ ] 无安全问题

## 🚨 问题响应流程

### 发现问题后的处理流程

```
发现问题
    ↓
问题分类（索引/抓取/安全/性能）
    ↓
评估严重程度（高/中/低）
    ↓
制定修复方案
    ↓
执行修复
    ↓
请求重新验证（如适用）
    ↓
跟踪修复状态
```

### 常见问题处理

#### 问题1：大量页面未被索引
**可能原因**：
- 页面质量低
- 内链不足
- 页面被 noindex
- robots.txt 屏蔽

**处理步骤**：
1. 检查页面的 `<meta name="robots">` 标签
2. 检查 robots.txt 配置
3. 确认页面有足够内链
4. 使用 URL 检查工具提交索引请求

#### 问题2：404 错误增加
**处理步骤**：
1. 导出 404 页面列表
2. 分析 404 来源（内链/外链/Sitemap）
3. 重要页面设置 301 重定向
4. 更新内链和 Sitemap
5. 使用 URL 移除工具（如需）

#### 问题3：Core Web Vitals 不达标
**处理步骤**：
1. 识别问题页面
2. 使用 PageSpeed Insights 诊断
3. 针对性优化（图片/JS/CSS）
4. 重新测试验证
5. 等待 GSC 数据更新
