## 二、执行流程

### Phase 1: 页面创建前检查

```
1. 确定目标关键词（主关键词 + 3-5个长尾词）
2. 分析SERP前10结果的内容结构
3. 规划H标签层级结构
4. 设计内链策略（链向哪些页面）
5. 确定CTA类型和位置
```

### Phase 2: 页面创建中检查

```
1. URL结构符合规范
2. Title/Meta按模板编写
3. H1唯一且包含主关键词
4. 内容首段包含主关键词
5. H2/H3合理嵌入长尾词
6. 图片Alt文本完整
7. 内链锚文本语义化
```

### Phase 3: 发布前终检

```
1. 运行Lighthouse检查
2. 验证结构化数据
3. 确认所有CTA可点击
4. 移动端预览测试
5. 内链跳转验证
```

## 三、常见问题与解决方案

### 问题1: Title被截断
**原因**: 超过60字符或像素宽度超限
**解决**: 
- 控制在50-55字符
- 主关键词放最前面
- 品牌名放最后用 `|` 分隔

### 问题2: 多个H1标签
**原因**: 组件嵌套或模板问题
**解决**:
- 使用项目的 `Heading` 组件
- 检查 `src/components/ui/heading.tsx`
- 全局搜索 `<h1` 确认唯一性

### 问题3: 内链数量不足
**原因**: 内容较短或未规划内链
**解决**:
- 使用 `InternalLinking` 组件
- 参考 `src/components/InternalLinking.tsx`
- 在FAQ区域增加相关链接

### 问题4: 图片缺少Alt文本
**原因**: 开发时遗漏
**解决**:
- 使用 `ImageOptimized` 组件强制Alt
- 参考 `src/components/ImageOptimized.tsx`
- Alt文本包含关键词但自然描述

## 四、质量评分标准

### A级页面（90-100分）
- ✅ 所有P0检查项通过
- ✅ 80%以上P1检查项通过
- ✅ Core Web Vitals全绿
- ✅ 结构化数据验证通过

### B级页面（70-89分）
- ✅ 所有P0检查项通过
- ⚠️ 60-80% P1检查项通过
- ⚠️ 部分性能指标需优化

### C级页面（<70分）
- ❌ 存在P0检查项未通过
- ❌ 需要立即修复
