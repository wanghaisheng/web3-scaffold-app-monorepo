#### 📋 **实施流程**
```
选择试点 → 实施修复 → 测试验证 → 确认成功 → 批量推广
    ↑                                                    ↓
    └─────────── 如果失败 ────────── 调试方法 ──────────┘
```

#### 🔄 **失败处理流程**
- **试点失败**: 深入调试找到真正问题
- **调整方法**: 尝试不同的解决方案
- **重新试点**: 用新方法再次试点

#### 🚨 **一定要避免的错误**
1. **范围扩大**: 试点时不要修改其他模块
2. **类型分散**: 不要使用分散的类型定义
3. **命令随意**: 不要使用非标准命令
4. **测试跳过**: 不要跳过测试验证步骤

，我应该修复原来的问题而不是创建简化版本。
