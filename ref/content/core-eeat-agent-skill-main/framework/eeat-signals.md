# EEAT Signals

Google E-E-A-T (Experience, Expertise, Authoritativeness, Trust) framework implementation guide.

> **Related**: [core-principles.md](core-principles.md) for CORE content optimization

---

## Overview

| Signal | Full Name | Core Question | Weight |
|--------|-----------|---------------|--------|
| **E** | Experience | Does author have real usage experience? | 25% |
| **E** | Expertise | Does author have professional knowledge? | 25% |
| **A** | Authoritativeness | Are cited sources authoritative? | 25% |
| **T** | Trust | Is content trustworthy? Are interests disclosed? | 25% |

---

## E - Experience

### Core Concept

Content should come from **actual usage experience**, not just information aggregation.

### Checklist

| ID | Check Item | Requirement | Priority |
|----|------------|-------------|----------|
| E01 | **First-Person Narrative** | Has "I tested...", "We found..." | ⭐⭐⭐ |
| E02 | **Test Data** | Provides own test data/screenshots | ⭐⭐⭐ |
| E03 | **Testing Duration** | States test period/sample size | ⭐⭐ |
| E04 | **Specific Scenarios** | Describes actual use cases | ⭐⭐ |
| E05 | **Pain Points** | Honestly shares problems encountered | ⭐⭐⭐ |
| E06 | **Before/After** | Changes/improvements after using | ⭐⭐ |

### E01 First-Person Narrative

**Requirement**: Use first person (I/We) to describe actual experience.

**Correct**:
```markdown
We tested 10 AI tools across 50+ samples to find the best
options. [Tool A] delivered 94% accuracy in our tests, completing
the task in just 2-5 seconds.

I've been using [Tool A] for my workflow for 3 months now.
The main advantage I found is...
```

**Wrong**:
```markdown
[Tool A] is a popular tool that many users find helpful.
According to online reviews, it has good accuracy.
[No first-person experience]
```

### E02 Test Data

**Requirement**: Provide specific test parameters and results.

**Format**:
```markdown
## How We Tested

**Test Parameters:**
- 50 samples across 8 categories
- Duration: 5-90 minutes each
- Languages: English, Spanish, Chinese
- Test period: January 2026

**Results:**
| Tool | Accuracy | Speed | Languages |
|------|----------|-------|-----------|
| Tool A | 94% | 2-5 sec | 100+ |
```

### E05 Pain Points

**Requirement**: Honestly describe product limitations and issues encountered.

**Example**:
```markdown
### Limitations

During our testing, we encountered a few issues:

- No real-time processing (must wait for completion)
- Occasional timeout on very large inputs
- Accuracy not 100% for complex cases

**Workaround:** For very large files, we recommend splitting
into smaller segments.
```

## E - Expertise

### Core Concept

Content should demonstrate **professional knowledge** with correct terminology and complete reasoning.

### Checklist

| ID | Check Item | Requirement | Priority |
|----|------------|-------------|----------|
| E01 | **Correct Terms** | Professional terminology used accurately | ⭐⭐⭐ |
| E02 | **Complete Reasoning** | Causal chains, no logical jumps | ⭐⭐ |
| E03 | **Scope Boundaries** | States applicable/non-applicable scenarios | ⭐⭐ |
| E04 | **Version Awareness** | Notes software/API versions | ⭐⭐ |
| E05 | **Alternatives** | Provides multiple solutions | ⭐⭐ |
| E06 | **Error Handling** | Explains what to do when things go wrong | ⭐⭐ |

### E01 Correct Terms

**Requirement**: Use professional terminology accurately, don't confuse concepts.

**Example - Term Distinctions**:
```markdown
Output vs Export vs Download:
- **Output**: The result generated by the tool
- **Export**: Converting to a specific format
- **Download**: Saving to local storage

API vs SDK vs Library:
- **API**: Interface for communication
- **SDK**: Complete development kit
- **Library**: Reusable code collection
```

### E03 Scope Boundaries

**Requirement**: Clearly state method/tool applicability.

**Example**:
```markdown
**Best For:**
- Users needing quick results
- Researchers analyzing content
- Content creators repurposing material

**Not Ideal For:**
- Real-time processing needs
- Private/restricted content
- Highly specialized domains
```

### E06 Error Handling

**Requirement**: Explain solutions for common problems.

**Example**:
```markdown
### Troubleshooting

**Problem: "No data available" error**
- Check if source is accessible
- Try a different input format
- Wait and retry (may be temporary issue)

**Problem: Incomplete output**
- Source may still be processing
- Some sources don't provide full data
```

## A - Authoritativeness

### Core Concept

Cite **authoritative sources** to support claims, using a tiered system to evaluate source quality.

### Checklist

| ID | Check Item | Requirement | Priority |
|----|------------|-------------|----------|
| A01 | **Authoritative Citations** | Cite tier1/tier2 sources | ⭐⭐⭐ |
| A02 | **Citation Tiers** | Use tier1/tier2/tier3 system | ⭐⭐⭐ |
| A03 | **Official Sources** | Pricing/features cite official docs | ⭐⭐⭐ |
| A04 | **Verifiable** | Citation links accessible | ⭐⭐ |
| A05 | **Timeliness** | Note citation access date | ⭐⭐ |

### Citation Tier System

| Tier | Source Type | Credibility | Examples |
|------|-------------|-------------|----------|
| **tier1** | Official docs/primary sources | Highest | Official help centers, API docs |
| **tier2** | Industry authority/major media | High | TechCrunch, Harvard Business Review |
| **tier3** | Community/user reviews | Reference | Reddit, G2 Reviews |

### TypeScript Implementation

```typescript
interface ExternalCitation {
  title: string;           // Source title
  url: string;             // Source URL
  tier: 'tier1' | 'tier2' | 'tier3';
  accessDate: string;      // YYYY-MM-DD
  description?: string;    // Source description
}

// Usage example
externalCitations: [
  {
    title: 'Official Documentation',
    url: 'https://example.com/docs',
    tier: 'tier1',
    accessDate: '2026-01-15',
    description: 'Official product documentation'
  }
]
```

## T - Trust

### Core Concept

Content should be **transparent and trustworthy**, with disclosed interests and public author info.

### Checklist

| ID | Check Item | Requirement | Priority |
|----|------------|-------------|----------|
| T01 | **Author Info** | Provides author/company info | ⭐⭐⭐ |
| T02 | **Contact Method** | Can reach author/company | ⭐⭐ |
| T03 | **Interest Disclosure** | Declares affiliate/samples/sponsorship | ⭐⭐⭐ |
| T04 | **Update Commitment** | Notes content timeliness | ⭐⭐ |
| T05 | **Fair Analysis** | Comparison content acknowledges competitor strengths | ⭐⭐⭐ |
| T06 | **Disclaimers** | Adds disclaimers when necessary | ⭐⭐ |

### T01 Author Info

**TypeScript Implementation**:
```typescript
interface AuthorInfo {
  name: string;
  bio: string;
  title?: string;
  avatar?: string;
  credentials?: string[];
  linkedIn?: string;
  twitter?: string;
  website?: string;
}

// Default author template
const DEFAULT_AUTHOR: AuthorInfo = {
  name: '[Team Name]',
  bio: 'Description of team expertise...',
  credentials: ['Credential 1', 'Credential 2']
};
```

### T03 Interest Disclosure

**Requirement**: Disclose any interests that could affect objectivity.

**Disclosure Types**:
```typescript
dataDisclaimer?: string;

// Examples
dataDisclaimer: '[Product] is our own product. We strive for objectivity but readers should be aware of this relationship.'

dataDisclaimer: 'This article contains affiliate links. We may earn a commission if you make a purchase.'

dataDisclaimer: 'We received a free sample for testing. This did not influence our review.'
```

### T05 Fair Analysis (Comparison Content)

**Requirement**: Comparison pages must fairly analyze competitors, acknowledging their strengths.

**Format**:
```markdown
## [Competitor]'s Strengths (Fair Analysis)

We acknowledge that [Competitor] excels in certain areas:

- **[Strength 1]**: Description
- **[Strength 2]**: Description
- **[Strength 3]**: Description

## [Competitor]'s Limitations

However, users often report these challenges:

- **[Limitation 1]**: Description
- **[Limitation 2]**: Description
```

### Disclaimer Templates

```typescript
// Pricing disclaimer
const PRICING_DISCLAIMER =
  'Pricing information is accurate as of the verification date. ' +
  'Prices may change. Please verify on official websites before purchasing.';

// Statistics disclaimer
const STATS_DISCLAIMER =
  'Statistics are from cited sources and may be subject to change. ' +
  'We recommend verifying current data from primary sources.';

// Testing disclaimer
const TESTING_DISCLAIMER =
  'Results are based on our testing methodology. ' +
  'Individual results may vary based on use case and requirements.';
```

## EEAT Priority by Content Type

| Content Type | Experience | Expertise | Authority | Trust |
|--------------|------------|-----------|-----------|-------|
| How-To Guide | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ |
| Tool Comparison | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| Product Review | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| Alternative Page | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| Use Case Page | ⭐⭐⭐ | ⭐⭐ | ⭐⭐ | ⭐⭐ |
| Best-Of Listicle | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| FAQ Page | ⭐⭐ | ⭐⭐ | ⭐⭐ | ⭐⭐ |
| Landing Page | ⭐⭐ | ⭐⭐ | ⭐⭐ | ⭐⭐ |
| Testimonial | ⭐⭐⭐ | ⭐ | ⭐⭐ | ⭐⭐⭐ |

### Priority Rationale

| Content Type | Why This Priority |
|--------------|-------------------|
| **How-To Guide** | Experience high: "We tested" narratives prove real usage. Expertise high: Correct terminology matters. |
| **Tool Comparison** | All high: Testing experience + technical knowledge + cited sources + fair analysis. |
| **Alternative Page** | Experience high: Must use both products to compare fairly. Trust high: Interest disclosure critical. |
| **Use Case Page** | Experience high: Specific scenarios require real-world examples. |
| **Best-Of Listicle** | Experience high: Testing methodology. Authority high: Verified pricing/features. |
| **FAQ Page** | Experience moderate: First-person answers add value. Low stakes content. |
| **Landing Page** | Balanced: Features matter more than deep expertise. Product knowledge sufficient. |
| **Testimonial** | Experience high: Real customer story. Trust high: Verification is the content's value. |

**Note**: Experience was adjusted higher for How-To Guide and Alternative pages because these content types benefit significantly from "We tested..." narratives per CORE principles.
