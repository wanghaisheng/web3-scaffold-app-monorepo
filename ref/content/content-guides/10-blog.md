# Blog 写作指南（guides / tools / growth / insights）

Blog 的目标是“教会读者做成一件事”或“帮助读者选一个工具/方案”。写法以首屏答案、可复查信息、真实经验为核心，避免模板化的长铺垫。

## 目标与适用

- guides：教流程、教方法（How to）
- tools：对比/评测工具（Best tools / Tool comparison）
- growth：增长与变现（更偏策略与可执行动作）
- insights：趋势/统计/行业观察（必须更强调引用与口径）

## 推荐结构（通用）

```markdown
# 标题（承诺明确、非标题党）

首段：150 字内直接给答案/结论（含推荐方案或结论句）

## Key Takeaways
- 5–7 条，带数字/条件/边界，不要口号

## 正文主体
按意图选择：方法分步 / 工具对比 / 策略拆解 / 数据解读

## FAQ
5–10 个问题，回答 2–4 句，必要时给出处

## 结尾
回扣开头结论 + 下一步怎么做（用“检查点/指标/行动”收尾）
```

## 写作要点

### 首段（C01）
- 直接回答标题承诺：推荐什么、为什么、适用谁、不适用谁。
- 避免“背景介绍 + 本文将…”。

### 方法/步骤（O06 + R06）
- 步骤用 1-2-3；每步给可执行动作与结果判定。
- 写“失败会怎样”和“怎么修”：至少 1 个常见坑 + 解决方式。

### 数据与引用（R01/R02）
- 任何数字都写清单位、时间范围、口径；重要统计给 tier1/tier2 链接与 accessDate。
- 工具价格/政策/规格必须来自官方页面或权威来源。

### EEAT（最低配）
- 至少一段“我们怎么测/怎么判断”的描述：样本、周期、对比维度三选一。
- 可以写克制的第一人称，但必须接证据与边界条件。

## Humanizer 检查（Blog 常见雷区）

- 不要把正文硬拆成 1/2/3 三段“看起来全面”的结构。
- 少用“**标题：**解释”型列表；能合并成句子就合并。
- 结尾不要写“未来光明/新时代/转折点”，改成下一步观察指标或操作建议。

## 验收清单（发布前 2 分钟）

- 首段 150 字内给出明确答案/推荐，并包含边界条件。
- Key Takeaways 5–7 条，至少 3 条带数字/条件/口径。
- 至少 1 处“我们怎么知道”的经验/方法说明。
- 至少 1 条权威引用（涉及价格/政策/统计时必须有）。
- 无套话开头、无三段式强迫症、无口号式结尾、无“粗体标题：解释”竖排列表。
