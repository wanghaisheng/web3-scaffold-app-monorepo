# Landing 写作指南（产品/能力落地页）

Landing 的目标是“让读者立刻理解这是做什么的、适合谁、下一步做什么”。它不是长篇科普文，不要写成 blog。

## 适用场景

- 单一产品/功能的介绍、定位、转化
- 读者已经有需求，来确认“你能不能解决”

## 推荐结构

```markdown
# 一句话价值主张（读者视角）

首屏：一句话说明做什么 + 谁用 + 结果是什么

## What it does
用 3–5 句把能力边界讲清楚（包含不做什么）

## Key features
4–6 个要点：每条写“能力 + 约束/条件/量化指标”

## How it works
3–5 步，写清输入/输出与成功判定

## Proof / Evidence
案例、指标、截图、引用、对外可核验信息

## FAQ
4–6 个高频问题（隐私、价格、兼容性、限制）

## CTA
明确下一步（试用/联系/下载/查看文档）
```

## 写作要点

### 首屏要“快”
- 避免故事化开场与行业背景；直接说你解决什么问题。
- 第一屏就说明适用人群与一个核心差异点。

### 证据要“能复查”
- 任何承诺都给口径：速度、准确率、兼容性、支持范围等。
- 引用外部信息时带 accessDate。

### Trust
- 若有定价/试用限制，提前说明。
- 若是自家产品，披露利益关系不是扣分项，藏着才扣分。

## Humanizer 检查（Landing 常见雷区）

- 少用形容词堆叠（seamless/intuitive/powerful）；用可验证描述替代。
- 不要把每个功能点写成“**标题：**一句话”；能合并成自然句就合并。
- 避免口号式收尾，用 CTA + 预期结果收尾。

## 验收清单

- 首屏一句话说清“做什么/给谁/结果是什么/不做什么”。
- Features 4–6 条，每条包含约束或量化信息。
- How it works 3–5 步，可复现实操路径。
- FAQ 覆盖限制与风险点，不只回答好听的问题。
