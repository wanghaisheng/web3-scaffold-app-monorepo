# 项目意图与设计目标

## 一、项目定位

本项目是一个**学习和参考性质的 monorepo 脚手架**，基于 OneKey app-monorepo 的架构学习而来。

### 核心目标

1. **学习 OneKey monorepo 的工程化实践**
   - 理解大型 Web3 项目的组织结构
   - 学习多端代码复用策略
   - 掌握模块化设计模式
   - 研究工程化工具链配置

2. **提供可复用的 Web3 开发脚手架**
   - 统一的多端开发框架
   - 可配置的业务模块
   - 标准化的开发流程
   - 完善的文档和示例

3. **支持快速业务落地**
   - 针对 Web3 领域的业务需求
   - 提供常见业务场景的模板
   - 降低新项目的启动成本
   - 加速产品迭代速度

## 二、学习重点

### 2.1 Monorepo 组织结构

重点关注以下方面：

- **包管理策略**：Yarn workspaces 的使用和最佳实践
- **依赖管理**：内部包的引用和版本控制
- **构建流程**：多端应用的统一构建配置
- **代码共享**：跨平台的代码复用机制

### 2.2 多端架构

学习 OneKey 如何支持多平台：

- **Web**：传统 SPA + Astro SSR
- **Mobile**：React Native 原生应用
- **Desktop**： Electron 桌面应用
- **Extension**：浏览器扩展

### 2.3 模块化设计

研究 OneKey 的模块划分：

- **UI 组件层**：`packages/components`
- **业务逻辑层**：`packages/kit`
- **后台服务层**：`packages/kit-bg`
- **核心功能层**：`packages/core`
- **共享工具层**：`packages/shared`

## 三、脚手架设计

### 3.1 可复用模块

抽象出以下可复用模块：

#### UI 组件库

- 基于 Tamagui 的跨平台组件
- 统一的设计系统
- 暗色/亮色主题支持
- 国际化支持

#### 区块链核心

- 多链钱包基础架构
- 交易签名和广播
- 余额查询和代币管理
- DApp 连接和交互

#### 业务逻辑模块

- 账户管理
- 交易历史
- 设置管理
- 通知系统

#### 开发工具

- 代码生成器
- 脚手架模板
- 开发调试工具
- 部署脚本

### 3.2 业务场景模板

针对常见的 Web3 业务场景提供模板：

#### 钱包类产品

- 多链钱包
- 硬件钱包配套软件
- Web 钱包
- 机构托管方案

#### DeFi 产品

- DeFi 聚合器
- NFT 交易平台
- Token 兑换
- Staking 平台

#### B2B 产品

- 嵌入式钱包 SDK
- 支付网关
- 企业资产管理
- DAO 治理工具

#### 工具类产品

- 区块链浏览器
- Gas Fee 工具
- 链上数据分析
- 跨链桥

### 5.4 可扩展性

- 支持插件化架构

## 六、与 OneKey 的关系

### 6.1 学习来源

本项目主要学习 OneKey 的以下方面：

- 工程化组织方式
- 技术栈选择
- 架构设计思路
- 开发流程规范

### 6.2 差异化

与 OneKey 的主要差异：

- **定位不同**：OneKey 是产品，本项目是学习脚手架
- **范围不同**：OneKey 聚焦钱包，本项目覆盖更广泛的 Web3 场景
- **品牌不同**：需要去除 OneKey 品牌标识
- **依赖不同**：部分依赖可能需要替换或 fork

### 6.3 品牌替换

如需完全去除 OneKey 品牌，参考 [品牌替换分析](brand-replacement-analysis.md)：

- 可替换：包名、代码引用、文档
- 不可替换：Git 外部依赖、硬件钱包 SDK

## 八、注意事项

1. **学习性质**：本项目主要用于学习和参考，不建议直接用于生产环境
2. **法律合规**：使用时需要注意相关法律法规，特别是金融和加密货币相关业务
3. **安全审计**：如用于生产环境，需要进行充分的安全审计
4. **品牌使用**：去除 OneKey 品牌前需要评估法律风险
5. **依赖维护**：部分依赖需要自行维护和更新
